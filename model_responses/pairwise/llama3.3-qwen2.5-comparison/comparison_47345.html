The PCI path of a USB port changing randomly can be caused by several factors.
Here are some possible explanations:

  1. USB Port enumeration: When you plug in a device, the operating system enumerates the USB port and assigns a unique identifier. If the device is removed and reinserted, the enumeration process may result in a different PCI path. This is because the operating system may assign a new identifier or reorder the existing ones.
  2. USB Hub behavior: If the USB port is connected to a USB hub, the hub's
    behavior can affect the PCI path. Some hubs may reset or reenumerate when a
    device is connected or disconnected, which can cause the PCI path to change.
  3. Power management: Modern operating systems often implement power-saving
    features, such as USB port suspension or power gating. When a device is not in
    use, the operating system may suspend the USB port to conserve power. When the
    device is reactivated, the port may be reinitialized, resulting in a new PCI
    path.
  4. USB driver issues: Problems with the USB driver or firmware can cause the
    PCI path to change. For example, if the driver is not properly handling device
    removal or insertion, it may cause the operating system to reenumerate the port.
  5. Hardware issues: Faulty or flaky hardware, such as a loose connection or
    a malfunctioning USB controller, can cause the PCI path to change randomly.
  6. BIOS or UEFI settings: Some BIOS or UEFI settings, such as USB port
    configuration or legacy USB support, can affect the PCI path. If these settings
    are changed or not properly configured, it may cause the PCI path to change.
  7. Operating System settings: Certain operating system settings, such as USB
    device removal policies or power management settings, can also impact the PCI
    path.
  8. USB device itself: Some USB devices may have issues with their own
    firmware or hardware, causing them to be reenumerated or change the PCI path
    when connected to the same port.

To troubleshoot this issue, you can try the following:

  1. Check the system logs: Inspect the system logs to see if there are any error messages or warnings related to the USB port or device.
  2. Use a different USB port: Try using a different USB port on the same
    system to see if the issue persists.
  3. Update USB drivers: Ensure that the USB drivers are up-to-date and
    compatible with the operating system.
  4. Disable power-saving features: Temporarily disable power-saving features,
    such as USB port suspension, to see if it resolves the issue.
  5. Reset the USB hub: If the issue occurs with a USB hub, try resetting the
    hub or replacing it with a different one.

If none of these troubleshooting steps resolve the issue, it may be helpful to

The PCI path of a USB port on a PC can indeed change, and this can be caused by
several factors. Here are some common reasons why the PCI path might change:

  1. BIOS/UEFI Firmware Changes: Updates to the BIOS/UEFI firmware can sometimes alter how USB ports are enumerated, leading to a change in the PCI path.

  2. Hardware Changes: Any changes to the hardware configuration, such as
    adding or removing components, can affect the PCI enumeration process.

  3. Driver Issues: USB drivers or other system drivers that manage device
    enumeration can cause changes if they are updated or corrupted.

  4. Operating System Updates: Updates to the operating system can also lead
    to changes in how USB devices are enumerated and assigned PCI paths.

  5. Power Management Settings: Power management settings can sometimes cause
    USB ports to be powered down and re-enumerated, which can result in a change in
    the PCI path.

  6. Physical Connection Issues: Loose or faulty USB cables, hubs, or ports
    can sometimes cause the system to re-enumerate the device, leading to a
    different PCI path.

  7. USB Port Multiplexing: Some systems use multiplexing to share USB hubs,
    which can cause USB devices to be assigned different paths depending on the
    current state of the system.

  8. System Restart: A system restart can sometimes cause USB devices to be
    re-enumerated, leading to changes in their PCI paths.

Troubleshooting Steps

  1. Check BIOS/UEFI Settings: Ensure that the BIOS/UEFI settings have not been changed, and update the firmware if necessary.

  2. Update Drivers: Make sure that all USB and system drivers are up to date.

  3. Check for Hardware Changes: Verify that no hardware has been added or
    removed recently.

  4. Review System Logs: Check the system logs for any errors or warnings
    related to USB devices or PCI enumeration.

  5. Test with Different USB Ports: Try using different USB ports to see if
    the issue persists.

  6. Disable Power Management: Temporarily disable USB power management
    settings to see if it resolves the issue.

  7. Reinstall USB Controllers: Uninstall and reinstall the USB controllers in
    the Device Manager to ensure they are properly configured.

  8. Reset BIOS/UEFI to Defaults: Resetting the BIOS/UEFI to default settings
    can sometimes resolve issues caused by misconfiguration.

If the issue persists after trying these steps, it may be helpful to consult the
documentation for your specific hardware or seek assistance from the
manufacturer's support.
